# AGENTS.md

## 1) 프로젝트가 해결하려는 것
코코파우는 사용자가 화장품을 살 때 "이미 내가 갖고 있나?"를 바로 확인하고, 유통기한(사용 가능한 날짜)을 놓치지 않도록 돕는 온라인 화장품 파우치 앱이다.

구성 이유(한 줄): 모든 구현 판단의 기준이 되는 문제 정의를 맨 앞에 둔다.

---

## 2) 최종 기술 스택과 고른 이유
### 2.1 확정 스택
- 앱 데이터 저장: Azure Cosmos DB for NoSQL
  - 고른 이유: 화장품 목록 같은 문서형 데이터(한 제품 = 한 데이터 묶음)를 저장/조회하기 좋고, 확장에 유리하다.
- 코드 저장/버전관리: GitHub
  - 고른 이유: 변경 이력 관리(버전관리), 백업, 협업, 자료 접근성이 좋다.
- 앱-DB 연결 API: Azure Functions
  - 고른 이유: 앱이 Cosmos DB를 직접 만지지 않고, 안전한 중간 계층(API)을 통해 읽기/쓰기 하게 만들어 보안과 유지보수가 쉬워진다.

### 2.2 현재 원칙
- 초보자 우선: 구현 난이도보다 이해 가능한 구조를 우선한다.
- 문서/예제 우선: 공식 문서와 튜토리얼이 많은 방식을 우선한다.

구성 이유(한 줄): 기술 선택의 근거를 명확히 남겨 이후 변경 논의를 쉽게 한다.

---

## 3) 만들 핵심 기능
- 인증/접근
  - 비회원 읽기 전용 모드
  - 로그인 사용자 데이터 백업/복원 흐름
- 화장품 등록
  - 직접 등록(제품명, 카테고리, 개봉일, 사용기한/만료일)
  - 선택 입력(브랜드, 메모, 사진)
  - 바코드 스캔으로 코드값 읽기 + 수동 입력 보조
- 내 파우치 조회
  - 등록된 제품 목록 확인
  - 제품 상세 보기
  - 제품 수정/삭제
- 유통기한 관리
  - D-day 배지(남은 날짜 표시)
  - 만료 임박 정렬
  - 만료 임박 목록/만료됨 목록
- 지표 이벤트
  - MAU(월간 활성 사용자 수)
  - 사용자당 등록 제품 수
  - 만료 임박 확인률
  - 만료 제품 교체 체크율

구성 이유(한 줄): 실제 개발 범위를 기능 단위로 고정해 우선순위를 잃지 않게 한다.

---

## 4) 이번엔 만들지 않을 것
- 커머스 기능(구매 연동, 결제, 리뷰)
- 관리자 기능
- iOS 출시(이번 단계는 Android 중심)
- 푸시 알림(다음 단계)
- 외부 바코드 상품 DB 자동 연동(다음 단계)
- Apple 로그인(이번 Android 단계에서는 제외, iOS 단계에서 반영)

구성 이유(한 줄): 하지 않을 범위를 먼저 고정해야 일정 지연과 범위 확장을 막을 수 있다.

---

## 5) 협업 규칙(초보자 친화)
- 사용자(프로젝트 오너)는 코딩 초보다.
- 설명은 쉬운 한국어로 작성한다.
- 전문 용어는 처음 나올 때 반드시 괄호로 짧게 뜻을 붙인다.
  - 예: API(앱과 서버가 대화하는 창구)
  - 예: 배포(앱을 실제 사용자에게 공개하는 작업)
- 한 번에 너무 많은 선택지를 주지 않는다.
- 코드 제안 시 "왜 필요한지"를 먼저 1~2문장으로 설명한다.
- 큰 작업은 작은 단계로 나눠 체크리스트로 제시한다.

구성 이유(한 줄): 학습 속도와 구현 성공률을 높이기 위해 커뮤니케이션 방식을 명시한다.

---

## 6) 이 파일 사용 규칙
- 개발 중 의사결정이 필요하면 AGENTS.md를 우선 참조한다.
- PRD와 충돌 시, 사용자 최신 지시를 우선하고 AGENTS.md를 즉시 업데이트한다.
- 스택/범위/규칙이 바뀌면 같은 날 변경 이력을 남긴다.

구성 이유(한 줄): 문서를 살아있는 기준서로 유지해 대화와 구현의 일관성을 지킨다.
