# 코코파우 PRD

## 1. 문서 정보
- 문서명: 코코파우 제품 요구사항 문서(PRD)
- 제품명: 코코파우
- 문서 목적: 기존 메모 기반 요구사항 정리
- 문서 상태: 확정(1차)
- 작성일: 2026-06-27
- 버전: v1.0
- 작성자/검토자: 작성자(PO), 검토자(개발 리드/디자인 리드)

## 2. 제품 개요
코코파우는 사용자가 보유한 화장품을 앱에서 확인하고, 유통기한(또는 사용기한)을 관리할 수 있도록 돕는 앱이다.

## 3. 배경 및 문제 정의
- 사용자는 화장품 구매 시 현재 보유 여부를 즉시 확인하기 어렵다.
- 화장품 유통기한을 모르고 사용하는 경우가 있으며, 이는 피부 트러블로 이어질 수 있다.

## 4. 목표 및 비목표
### 4.1 제품 목표
- 보유 중인 화장품 목록을 외부에서도 쉽게 확인할 수 있게 한다.
- 유통기한 관리를 통해 교체를 돕고 트러블 발생 가능성을 줄인다.

### 4.2 비목표
- 구매 연동, 결제, 리뷰 등 커머스 기능 제공
- 관리자 기능 제공

## 5. 대상 사용자
- 화장품을 좋아하는 사람들
- 사용자 특성(기본안): 초보 사용자부터 고관여 사용자까지 포함
- 주요 사용 상황(기본안):
  - 오프라인/온라인 구매 직전 보유 여부 확인
  - 집에서 보유 제품 정리 및 만료 임박 제품 점검

## 6. 릴리스 전략 및 범위
### 6.1 1차 출시(MVP)
- 플랫폼: Android
- 언어/지역: 한국어, 대한민국
- 유통기한 관리 방식: 앱 내 표시 중심(D-day 배지, 만료 임박 정렬)
- 로그인: 구글 로그인 구현(Android), 비회원 읽기 전용 모드

### 6.2 2차 출시(확장)
- 플랫폼: iOS 확장
- 기능: 푸시 알림 추가, 외부 바코드 DB 연동 검토
- 로그인: iOS 확장 시 애플 로그인 제공

## 7. 사용자 흐름 및 화면
### 7.1 주요 사용자 흐름
1. 시작 화면에서 앱 이름 "코코파우"를 본다.
2. 시작 화면 터치 시 로그인 화면으로 이동한다.
3. 로그인 후 홈(메인) 화면으로 이동한다.
4. 메인 화면에서 등록/조회/유통기한 관련 탭을 사용한다.
5. 메인 화면의 특정 항목 선택 시 상세/편집/목록 화면으로 이동한다.

### 7.2 화면 목록
- 시작 화면: 앱 이름 표시, 터치 시 로그인 화면 이동
- 로그인 화면: 로그인 수행 후 홈 이동
- 메인 화면(탭): 내 파우치, 등록하기, 유통기한 임박, 설정
- 부수적 화면: 제품 상세, 제품 수정/삭제, 만료 임박 목록, 만료됨 목록
- 화면 상태 정의(기본안):
  - 진입 조건: 비회원은 조회 화면 진입 가능, 백업/복원 진입 시 로그인 유도
  - 빈 상태: 등록 제품이 없으면 "아직 등록된 화장품이 없어요" + 등록하기 CTA 노출
  - 에러 상태: 네트워크/저장 실패 시 토스트 또는 다이얼로그로 재시도 액션 제공

## 8. 기능 요구사항
### 8.1 인증 및 계정
- 로그인 방식: 구글 로그인, 애플 로그인
- 비회원 사용: 가능(읽기 전용)
- 로그인 필수 시점: 앱 초기 진입은 비로그인 허용, 데이터 백업/복원 또는 기기 변경 시 로그인 유도
- 결정 사항: 1차(Android)는 구글 로그인 중심, 애플 로그인은 2차(iOS)

### 8.2 화장품 등록
- 사용자는 화장품을 등록할 수 있어야 한다.
- 등록된 화장품은 메인 화면에서 확인 가능해야 한다.
- 입력 항목
  - 필수: 제품명, 카테고리, 개봉일, 사용기한(또는 만료일)
  - 선택: 브랜드, 메모, 사진

### 8.3 등록 방식
- 직접 등록 제공
- 바코드 스캔 등록 제공
- 바코드 처리 정책
  - 1차: 바코드 코드값 인식 + 사용자 수동 입력 보조
  - 2차: 외부 상품 DB 연동 검토

### 8.4 보유 목록 조회
- 사용자는 현재 보유 중인 화장품 목록을 앱에서 확인할 수 있어야 한다.

### 8.5 유통기한 관리
- 앱은 교체 시점을 인지할 수 있도록 유통기한 관리 기능을 제공해야 한다.
- 1차: D-day 배지 표시, 만료 임박 정렬
- 2차: 푸시 알림 추가

### 8.6 카테고리 체계
- 기본 카테고리: 스킨케어, 메이크업, 클렌징, 선케어, 헤어/바디, 기타

### 8.7 사용기한 기준
- 기본 기준: 개봉일 기준
- 입력 방식: 개봉일 입력 + 만료일 직접 입력 또는 D-day 표시용 날짜 입력

### 8.8 교체 완료 추적
- 교체 완료 이벤트: 제품 상세에서 "교체 완료" 버튼 클릭
- 집계 규칙: 단순 삭제는 교체 완료로 집계하지 않음

## 9. 비기능 요구사항
### 9.1 성능
- 앱 실행 후 3초 이내 메인 진입
- 제품 목록 100개 기준 1초 이내 표시

### 9.2 보안/개인정보
- 최소 수집 원칙 적용(로그인 정보 + 제품 정보)
- 비밀번호 직접 저장 없음(소셜 로그인 사용)
- 개인정보 처리방침 명시

### 9.3 접근성
- 글자 크기 확대 대응
- 버튼 터치 영역 확보
- 명확한 색상 대비

### 9.4 지역화
- 1차: 한국어, 대한민국

## 10. 데이터 및 이벤트 요구사항
### 10.1 데이터 필드
- 제품: 제품명, 카테고리, 개봉일, 사용기한/만료일, 브랜드(선택), 메모(선택), 사진(선택)
- 이벤트: 교체 완료 이벤트 필드

### 10.2 측정 이벤트
- MAU
- 사용자당 등록 제품 수
- 만료 임박 확인률
- 만료 제품 교체 체크율
- 이벤트 정의(기본안):
  - MAU: 해당 월 `app_open` 또는 `home_view`를 1회 이상 발생시킨 고유 사용자 수
  - 사용자당 등록 제품 수: 해당 월 말 기준 전체 등록 제품 수 / 해당 월 MAU
  - 만료 임박 확인률: (`near_expiry_list_view` 발생 사용자 수) / (만료 임박 제품 보유 사용자 수)
  - 만료 제품 교체 체크율: (`replacement_completed` 발생 건수) / (같은 기간 만료된 제품 건수)
  - 집계 주기: 일 단위 적재, 주간 모니터링, 월간 리포트 확정

## 11. 성공 지표
- 월간 활성 사용자(MAU)
- 사용자당 등록 제품 수
- 만료 임박 확인률
- 만료 제품 교체 체크율
- 초기 목표값(런칭 후 3개월):
  - MAU 1,000 이상
  - 사용자당 등록 제품 수 8개 이상
  - 만료 임박 확인률 60% 이상
  - 만료 제품 교체 체크율 30% 이상

## 12. 의존성 및 정책
- 개인정보 처리방침 초안 작성 필요
- 이벤트 정의서(MAU, 등록 제품 수, 만료 임박 확인률, 교체 체크율) 작성 필요
- 백업/복원 정책(기본안):
  - 비회원: 기기 로컬 저장만 사용(백업/복원 미지원)
  - 로그인 사용자: 계정 기반 클라우드 백업/복원 지원
  - 동기화 시점: 로그인 직후, 앱 시작 시, 데이터 변경 시
  - 충돌 처리: 최근 수정 우선(Last Write Wins)
- 바코드 스캔 SDK(기본안): Google ML Kit Barcode Scanning(Android 1차)

## 13. 리스크 및 오픈 이슈
- 1차 Android에서 애플 로그인을 제외하면 계정 정책/사용자 기대와의 차이가 생길 수 있음
- 바코드 1차 방식은 자동 완성 정확도 한계가 있을 수 있음
- 만료 임박 기준일: 하루 전, 3일 전, 일주일 전
- 비회원 모드 조회 범위(기본안):
  - 허용: 목록 조회, 상세 조회, 정렬, 필터
  - 제한: 등록, 수정, 삭제, 백업/복원

## 14. 수용 기준(초안)
- 사용자는 제품 필수 항목을 입력해 등록할 수 있어야 한다.
- 등록된 제품은 메인 화면에서 확인 가능해야 한다.
- 제품 목록은 만료 임박 기준으로 정렬 가능해야 한다.
- 제품별 D-day 정보가 표시되어야 한다.
- 비회원은 읽기 전용으로 목록 조회 가능해야 한다.
- 예외 처리 및 실패 안내(기본안):
  - 필수값 누락 시: "필수 항목을 입력해주세요"
  - 바코드 인식 실패 시: "바코드를 인식하지 못했어요. 직접 입력해 주세요"
  - 저장 실패 시: "저장에 실패했어요. 다시 시도해 주세요"
  - 네트워크 오류 시: "네트워크 연결을 확인한 후 다시 시도해 주세요"
  - 권한 거부(카메라/사진) 시: "권한 허용 후 기능을 사용할 수 있어요"
